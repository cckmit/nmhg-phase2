claim details

When you double click on a claim number in any folder or in the search results, the claim opens in the details page in a new page, <claim number>. Depending on the folder from which you are viewing the detail page, you are allowed to add/edit data on this page. If you have opened the detail page from a read-only folder, then you cannot perform any add/edit on this page.

 

Note: The data shown in the claim detail page depends on the current status of the claim. Hence you may or may not see all sections and all fields mentioned below for some claims.

 

The claim detail page is divided into various sections for ease of navigation. You can view the following sections in this page: Processing Errors, Notifications, Claim Details, Dealer/Servicing Location, Truck Details, Failure Information, Service Information, Components Replaced, Travel Details, Miscellaneous, Comments, History, Part Return History, Manage Documents, Payment Detail and Credit Memo History. The sections shown in the detail page depends on the folder from which you are viewing the page.

 

The fields under some sections on this page may be hidden. To view the fields, click on the section header. This will expand the section showing all fields under it. Likewise, clicking on the section header of an expanded section collapses it hiding all fields under it.

 

Errors

This section displays all processing rule failures that occurred on this claim. These are the rules whose failure led to the claim being assigned to you for manual processing. You can see the description of each failed rule here.

History

This section displays the audit trail of the claim. It shows the following details for each of the update that happened on the claim in a tabular format:

  • Date: The date when the update happened. Clicking on the date value displays the claim detail page that shows the details of the claim, as corresponding to this update, without the Claim History section.

  • Claim State: The status of the claim after that update happened.

  • User: Name of the user who performed the update.

  • User Type: The type of user such as internal user, system user, dealer etc.

  • External Comments: Any external comments specified for the claim during that update.

  • Internal Comments: Any internal comments specified for the claim during the update.

  • Decision: The decision taken on that claim.

Data shown in this section is sorted in descending order.

Claim Details

This section shows the following details about the claim:

  • Claim Number: The claim number generated while filing the claim.

  • Claim Type: Kind of claim. The claim can be of type machine, parts or field product improvement.

  • Claim Status: The status of the claim.

  • Recovery Claim State: This field appears only for recovery claims. It displays the recovery claim state of the claim.

  • Request SMR: If the dealer had sent this claim to the DSM for a review, then this check-box will be checked.

  • Authorization Received: This field displays Yes if you have got authorization for submitting this claim by selecting the checkbox Authorization Received.

  • NCR: This field displays Yes if the Processor has filed an NCR claim.

  • NCR With 30 Days: This field displays Yes if the Processor has filed an NCR With 30 Days claim.

  • SMR Reason: This field will be displayed only if the checkbox for Request SMR is selected. It displays the reason provided by the dealer for submitting the claim to the DSM for review.

  • Dealer: Name of the service provider who filed this claim.

  • Date of Claim: Date when this claim was submitted.

  • Work Order Number: The work order number of the claim.

  • Date of Failure: Date when the truck failed.

  • Repair start Date: Date when the truck was repaired.

  • Repair start Date: Date when the truck was repaired.

  • Truck Serial Number: This field displays the serial number of the truck on which you have submitted the claim.

  • Claim Processed As: The status of the claim after its processing. For e.g when the dealer files the claim and there is no warranty for that machine and the claim is out of warranty then this field displays "Out of Warranty".

  • Return Part Status: The return part status of the returned part.

  • Source Warehouse: The source warehouse name of the returned part.

Dealer/Servicing Location

This section provides you information regarding the servicing location of the customer.

Truck Details

This is a read-only section. For a machine claim, this section displays the following details about the truck on which this claim was submitted:

  • Serial Number: Serial number of the truck. This is a hyperlink to the truck details page of the truck. For further details on the truck details page, click here.

  • Item Number: Item number of the truck.

  • Item Description: Item description of the truck.

  • Product: Product name of the truck.

  • Model: Model of the truck.

  • Hours On Truck: The number of hours the truck was in service till the date of failure.

  • Description: A short description of the truck. If the description is truncated, mouse over on the text to view the entire description.

  • Startup Date: Date when this truck was delivered to the end customer.

  • Warranty Start Date: Date when the warranty coverages commenced on this truck. This is the earliest start date among all policies associated with this truck.

  • Warranty End Date: Date when this truck will be out of warranty. This is the latest end date among all policies associated with this truck.

For a parts claim, the following details about the part on which this claim was submitted, are shown here:

  • Part Serial Number: Serial number of the part on which you are filing a claim.

  • Part Number: Part number of the part.

  • Product: Product name of the truck in which this part was installed.

  • Model: Model of the truck in which this part was installed.

  • Description: A short description of the part. If the description is truncated, mouse over on the text to view the entire description.

  • Date of Installation: Date when this part was installed in the truck.

  • Warranty Start Date: Date when the warranty coverages commenced on this part. This is the earliest start date among all policies associated with this part.

  • Warranty End Date: Date when this part will be out of warranty. This is the latest end date among all policies associated with this part.

  • Truck Serial Number: Serial number of the truck in which this part was installed. This will a hyperlink to the truck details page of that truck. For further details on the truck details page, click here.

Failure Information

You can see the following details about the failure that occurred in the truck/part:

  • Causal Part Number/Causal Part Serial Number: Causal part details as entered by the dealer. If the causal part was serialized, then you can see the serial number of the part here. If it was a non-serialized part, then you can see the part number of that part here.

  • Causal Part Description: A short description of the causal part.

  • Fault Location: Failure location as entered by the dealer.

  • Fault Found: Fault identified in the truck/part.

  • Failure Detail: Failure found in the truck/part.

Service Information

In this section, you can see the following details about the services performed by the dealer to correct the fault in the truck/part:

  • Job Code: The service procedure entered by the dealer. A mouse over on the icon alongside the job code will show a short description of the service procedure.

  • Service Technician: Name of the technician who performed the above mentioned labor.

  • Standard Labor Time: The standard labor time required to perform the above mentioned labor.

  • Additional Labor Time: Any additional labor specified by the dealer.

  • Reason for Additional Labor Time: Reason for using additional labor, as specified by the dealer.

  • Over Time Labor Split: Overtime labour, if any.

  • Labor Hours(s): The number of hours the labor performed beyond the normal working hours.

  • Reason for Over Time Labor Hour(s): Reason for performing the overtime labor hours.

Components Replaced

In the Parts Replaced sub-section, you can view three sub-sections: OEM Parts Replaced Installed,  Miscellaneous Expense & Outside Services and Miscellaneous Parts.

OEM Parts Replaced Installed

Installed Parts

  • Part Number: Part number of the faulty part.

  • Quantity: Number of parts of the given specification that were installed.

  • Unit Price: Unit price of the part.

  • UOM: Unit of Measurement of the part.

  • Unit Cost Price: Unit cost price of the part.

  • Description: A short description of the part.

  • Failure Report: Failure report for the part.

Removed Parts

  • Part Number: Part number of the faulty part that was removed from the truck.

  • Quantity: Number of parts of the given specification that were replaced.

  • Unit Price: Unit price of the part.

  • UOM: Unit of Measurement of the part.

  • Unit Cost Price: Unit cost price of the part.

  • Description: A short description of the part.

  • Return?: Select this checkbox to enable part return.

  • Return To: Select the desired option from the available options.

  • Return Details: The following Return Details are displayed:

  • Return Location: Enter a return location.

  • Payment Condition: You can select any payment condition out of four payment conditions appearing in the dropdown:

    • Pay without part return: If this payment condition is set for a part during part return configuration, then payment is released to dealer if claim is accepted ignoring the part return status.

    • Pay on part return: If this payment condition is set for a part during part return configuration, then payment is released to dealer if part is received at NMHG lab and claim is accepted.

    • Pay on part inspection: If this payment condition is set for a part during part return configuration, then payment is released to dealer if part is received, inspected and accepted at NMHG lab and the claim is accepted.

    • Pay On vendor Acceptance: If this payment condition is set for a part during part return configuration, then payment is released to dealer if the corresponding supply recovery claim is accepted by the supplier/vendor.

  • Due Days: Enter the number of due days.

  • RGA Number: Enter the RGA number.

  • Dealer Pickup Location: Select the dealer pickup location from the dropdown.

Miscellaneous Expense & Outside Services

  • Description: A short description of the replaced part.

  • Quantity: Number of parts of the given specification that were replaced.

  • Unit Price: Cost price of the replaced part.

  • Invoice: Supporting invoices attached by the dealer. To view the invoice, click on its link. The Warranty Management system allows you to save the invoice to your desktop so that you can view it.

Miscellaneous Parts

In this sub-section, you can view the following details about any outside labor or part used by the dealer for repair:

  • Part Number: Part number of the faulty part that was removed from the truck.

  • Description: A short description of the replaced part.

  • Quantity: Number of parts of the given specification that were replaced.

  • Unit Price: Cost price of the replaced part.

  • UOM: Unit of Measurement of the part.

  • Threshold Quantity: The threshold quantity of the part.

Travel Details

In this section, you can enter the travel information of the dealer:

  • Travel Distance: This field is autopopulated based on Google maps calculation.

  • Travel Location: Click on the Google maps link. This pre populates From address as Dealer location or remote technician location and To address as customer address. Edit the To Address,  if desired and click Submit.

  • Additional Travel Distance: Enter the additional travel distance, here.

  • Reason for Additional Travel Distance: Enter your reason for the additional travel distance.

  • Travel Trips: Enter the number of travel trips to the specified location.

  • Travel Hours: This field is autopopulated based on Google maps calculation.

  • Additional Travel Hours: Enter the additional travel hours, here.

  • Technician: Use the drop-down to specify the technician who performed this labor. This field is optional.

Miscellaneous

Here, the section Incidentals allows you to enter the other related expenses such as Item Freight and Duty, Parking and Toll expense, Meals, etc. To attach an invoice with the expense amount, click on the link Attach invoice.

 

Comments

Enter your comments for Condition Found, Work Performed, Claim Notes and Probable Cause.

part return history

This section displays the part return status details. The Processor can see the part return history details sections of a claim to know about the parts receive or inspect status. The Dealer can also view the part return history. It shows the following details for each of the update that happened on the part in a tabular format:

  • Quantity: The quantity of the parts to be returned.

  • Return Location: The return location of the part after that update happened.

  • BIN: The name of the bin.

  • Date: The date when the update happened.

  • Status: The status of the part after that update happened.

  • User: Name of the user who performed the update.

  • Comments: Any comments specified for the part during that update.

  • Action Performed: The action performed on the part.

Manage Documents

In this section you can attach binary documents to support any of expenses and claims or the nature of failure in the truck.

The following details are shown as part of this section:

  • File Name: This field displays the name of the document which you want to attach.

  • Document Type: To attach a document, select a document type from the Document Type dropdown and click on the button. A pop-up window is shown. Click on the Browse button to browse through your system for the document. You can attach up to 20 documents for a claim. After you have selected the document type and mentioned the exact location of the documents, click Attach. All selected documents will be attached to the claim. To delete an entry, click on the option.

  • Share With Supplier: Select this checkbox to share the attached document with the supplier.

  • Share With Dealer: This checkbox appears only for recovery claims for the Suppliers and Recovery Processors. They can select this checkbox to share the attached document with the dealer. Thus, only after the Supply Recovery Processor consent , these documents are uploaded to the warranty claim and are present for dealer view.

Note: The List of documents appearing in the Document Type dropdown is a managed list of values by the Admin.

Recovery Claim History

This section appears only for recovery claims. It displays the details of all recovery claims that have been filed so far for this truck.

  • Recovery Claim state: Current status of the recovery claim.

  • User: Login details of the user for this recovery claim.

  • Comments: Comments mentioned while filing a recovery claim.

  • Accepted/Rejected/Cannot Recover Reason: Reason associated with the acceptance, rejection or cannot be recovered for this claim.

  • Recovered Amount: Amount to be recovered under the claim.

  • Date: Date on which the recovery claim was filed.

Payment Details

In this section you can see complete details of the claim expenses broken down by different cost categories. The following details are shown as part of this section:

  • Cost Category: Name of the cost category.

  • Amount Asked: Amount asked by the service provider for this cost category.

  • Current Amount: Current claim amount for this cost category. this amount may or may not be equal to that claimed by the dealer.

For example, if you changed the labor hours for this claim, then the amount shown in the Current Amount field will be different than the amount shown in the Amount Asked field for Labor Hours cost category.

  • Percentage to Accept: The percentage of the total amount for this cost category accepted by you.

Credit Memo History

This section displays all the credit/debit memo generated for this claim. You can find the following details for each memo, sorted in descending order of the memo date, in a tabular format:

  • Credit Memo Number: Unique number assigned to this credit memo.

  • Credit Memo Date: Date when the credit memo was generated.

  • Tax Amount: Amount associated with this credit memo.

  • Paid Amount: Amount paid for this credit memo.

  • Credit/Debit Flag: Kind of memo. The memo can be of type credit or debit.

Note: The Credit Memo section is blank for NCR Claims.
