Processor review

When a claim is first assigned to you for manual adjudication, it appears in your Processor Review folder. You can make any changes to the claim details submitted by the service provider, except the claim type, the equipment details and SMR Request/Reason data entered by the service provider.

 

To view all claims assigned for processor review,

  • Click on the Processor Review link in the Claims section under Tasks column on your Home page.

  • Click on the Claims tab on the left-hand pane. The tab will open to reveal the folders under it. Click on the Processor Review folder.

Either of the above actions takes you to the Processor Review page. On this page you can see the Processor Review table that lists all claims assigned to you for manual adjudication. For further information on navigating through the table, see Lists.

 

You can see the following details for each claim in the Processor Review table:

  • Claim Number: Reference number of the claim.

  • Claim Type: Kind of claim. A claim can be of type machine, parts or campaign.

  • Dealer: Name of the service provider who filed this claim.

  • Fault Code: Failure code associated with this claim.

  • Causal Part: Part serial number or part number of the part which caused the failure in the equipment.

  • Failure Date: Date when the equipment failed.

  • Repair Date: Date when the equipment was repaired.

You can view the details of any claim by clicking on it. Double clicking on a claim will open it in a new page, <claim number>. The claim detail page for a claim in this folder, is divided into various sections for ease of navigation: Processing Errors, Notifications, Claim History, Claim Header, Equipment Details, Failure Information,, Service Information, Components Replaced, Miscellaneous, Manage Documents, Payment Details and Actions. The fields under some sections on this page may be hidden. To view the fields, click on the section header. This will expand the section showing all fields under it. Likewise, clicking on the section header of an expanded section collapses it hiding all fields under it.

 

For further information on Claim History, Claim Header, Equipment Details, Failure Information, Service Information, Components Replaced, Miscellaneous, and Manage Documents click here.

 

Processing Errors

This section displays all processing rule failures that occurred on this claim. These are the rules whose failure led to the claim being assigned to you for manual processing. You can see the description of each failed rule here.

 

Components Replaced

In the Parts Replaced sub-section you can add, remove or update parts and also specify return locations for the newly added parts. For each of the part, you can see the following additional details in a tabular format:

  • Part Return Location: The location to where the part must be returned. If this part is already marked for return then this field will show the current part return location and show a “-” if it is not to be returned.

  • Due Date: The date by which the part is due for return.

Payment Details

In this section you can see complete details of the claim expenses broken down by different cost categories. The following details are shown as part of this section:

  • Instructions: You can see some instructions shown here. The instruction is "Please note that Percentage Accepted takes effect only if you decide to accept this claim."

  • Payment Details: The following columns are displayed here:

  • Cost Category: Name of the cost category.

  • Amount Asked: Amount asked by the service provider for this cost category.

  • Current Amount: Current claim amount for this cost category. this amount may or may not be equal to that claimed by the dealer.

For example, if you changed the labor hours for this claim, then the amount shown in the Current Amount field will be different than the amount shown in the Amount Asked field for Labor Hours cost category.

  • Percentage to Accept: This is an editable field and you can enter the percentage of the total amount for this cost category that will be paid to the dealer for this claim. You can apply the percentages either at the totals of each individual cost category or at the total claim amount level and not at both levels.

  • Accepted Amount: The final amount that is calculated after applying the accepted percentage entered by you.

Actions

You need to enter comments in the following fields for a claim in the Processor Review folder:

  • Internal Comments: The comments that you enter in this field are visible only to other OEM users.

  • Comments to Dealer: The comments that you enter in this field are visible to both the OEM users as well as to the dealer who filed this claim. It is mandatory for you to enter comments in this field in case you are sending the claim back to the dealer for further information or if you are denying the claim.

The following actions are allowed on a claim in this folder:

  • Put on Hold: Select this action if you want to put this claim on hold. This claim will then be moved to your On Hold folder. You can do any changes to the claim and take similar actions on it.

  • Request Information from Dealer: Select this action if you need more information from the dealer on this claim. The claim will be forwarded to the dealer who filed it and it will be moved to the Forwarded inbox of the dealer. Incase if external user, the claim would move to the 'Forwarded Externally' inbox and incase of internal users, it would move to 'Forward Internally' inbox.

  • Request for Advice: Select this action if you want to send the claim internally to an authorized OEM user for an advice. Use the drop-down field following this action to select an internal user to whom you want to forward this claim for advice. When the OEM personnel or the dealer reply to your claim, the response is shown in the 'Replies' inbox.

  • Transfer to: Select this action to transfer the claim to another processor for manual adjudication. Use the drop-down field following this action to select an authorized processor or warranty administrator to whom you want to transfer this claim. The claim will now appear in the 'Transferred' inbox of the processor to whom the claim has been transferred.

  • Accept: If you want to accept this claim, then select this radio button. Use the drop-down field following this action to select an acceptance code stating the reason for your accepting this claim.

  • Deny: If you want to deny this claim completely, then select this radio button. Use the drop-down field following this action to select a rejection code stating the reason for denying this claim.

  • Send for CP Review: Select this action to send the claim to CP advisor for review. Use the drop-down field following this action to select a CP advisor to whom you want to send the claim for review.

Note: Once a claim is moved from the Processor Review folder to any other folder, it will not come back to this folder.

 

You can see the following action buttons on this page:

  • Cancel: Click on this button if you want to close this page without saving any details entered by you for this claim.

  • Validate: Click on this button to run the claim through the processing rules for validation and verification. The Validate pop-up window will be shown. Click here to know more about the Validate window.

A success message will be shown confirming your action. The message will hide itself and the current page will be closed. The claim will now be routed according to the decision taken by you. If you added any new parts in the Components Replaced section, then the dealer be accordingly notified about it.

 

Thus, when a claim is sent for review, the processor receives it in his/her "processor review" inbox. The system displays an option "approve and transfer to next user" that indicates that the user approves the claim but wishes to transfer it to a user who has the authority to approve the claimed amount. The system then sends the claim into the next user's pending authorization inbox.
