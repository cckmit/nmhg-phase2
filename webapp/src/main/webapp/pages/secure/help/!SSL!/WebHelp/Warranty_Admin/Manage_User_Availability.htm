MANAGE User AVAILABILITY

When a claimant files a claim, it undergoes automatic claim processing. In the automatic claims processing engine, a claim is validated against predefined rules. If the claim passes all the processing rules during automatic claims processing, the claim amount is calculated and paid to claimant automatically. But if the claim fails in the automatic claims processing engine, a manual review is needed to process it. This assignment of claims to the processor depends upon the availability of users.

 

There are different user roles for which the administrator sets the availability:

  • Processor

  • District Service Manager (DSM)

  • DSM Advisor

  • Recovery Processor

  • CP Advisor

To view the user availability, click on Manage User Availability tab under the Warranty Admin tab. Click on the individual roles to view the Manage user availability page which gets pre populated with the users. Some users are common to the roles. For e.g. a user may be a District Service Manager and a Processor at the same time. In such case the particular user will be available in the list of both the roles (DSM and Processor).

 

The user page is divided into three column “User”, “Availability” and “Default User”.

 

A list of processors are displayed under the processor inbox. Click on one of the processor in the detail page and you can view the User Availability Information page. If you mark the processor's availability as 'Yes', then this processor becomes available for the claim assignment. If the administrator marks its availability as 'No', no claims are assigned to these processors because of its unavailability.

Similarly the administrator can open the DSM, DSM Advisor, Recovery Processor and CP Advisor inbox and mark their availability as 'Yes' or 'No' as well.

 

If the users become unavailable or due to some reason they are not able to process the claim then the claim will automatically get assigned to the default user. The administrator can set the user as a default user by selecting 'Yes' against the Default User under the User Availability Information page.
