MANAGE LOA SCHEMES

The level of authority (LOA) is the authority given to a user to accept up to a certain amount on a claim. The warranty administrator defines LOA schemes that in turn get used to route claims through the levels of authority defined in the scheme. The system assigns a level number to each user in the hierarchy which helps in transferring claims to other users in other LOA schemes.

 

To view/create a LOA scheme,

  • Click on Manage LOA Schemes link on the left pane under the Warranty Admin tab. The system displays a listing pane that displays all the LOA schemes already present in the system.

  • Click on Create LOA Scheme button in the listing toolbar and enter the following details under the New LOA Scheme page.

  • LOA Scheme Code: A unique code that can be assigned by the admin to the scheme.

  • LOA Scheme Name: The name given to the scheme by the warranty administrator that would be used to identify the scheme.

  • LOA Scheme Description: A detailed description of the LOA scheme being created or updated.

  • Specify the authority levels by clicking on the button. This section captures the different authority levels and corresponding approval limits for the scheme in all currencies supported by the system.

  • Level: The Level number.

  • Level Name: The name given to the level – such as Regional Director, District Sales Manager.

  • Login: The login of the user at that level.

  • Approval Limit: The amount limit assigned to each level. This amount is set up for each currency supported by the system.

  • Click Save.

You can add multiple levels by clicking on the button. To delete a row, click button corresponding to that row.

 

Thus, when a claim is sent for review, the processor receives it in his/her "processor review" inbox. The system displays an option "approve and transfer to next user" that indicates that the user approves the claim but wishes to transfer it to a user who has the authority to approve the claimed amount. The system then sends the claim into the next user's pending authorization inbox.

  • If the user accepts the claim, but the current amount on the claim is greater than the user's level of authority, the system displays an error message:

"The amount on the claim is greater than your approval limit – Can not accept the claim. If you approve, please use the approve and transfer to next LOA user option."

  • If the user selects this option, but there is no LOA scheme being followed (because the rule was set up to be sent to the user directly and not through a LOA scheme), then the system displays an error message:

"Can not perform action - No LOA scheme to follow."

Note: All the users in the LOA scheme are processors and any user who wishes to be a part of the LOA scheme should have the processor role assigned to him/her.

When the claim is moving up the LOA chain and there are multiple users at the same level, the claim can be assigned to any of those users in a load-balancing manner.
