PaYMENT DEFINITION

The Warranty Management application allows you to define various modifiers for each component of the claim amount. You can apply these modifiers to their respective components at various levels of claim payment computation. Each such payment definition can be applied to a policy or a group of policies. The definitions thus created are applied while validating a claim, covered under the applicable policy, and calculating the reimbursement amount for each component of the claim amount.

 

You can assign a set of modifiers to a policy or a group of policies of a specified claim type. The policies are grouped together by a common label (for further information, refer Labels). You can assign the following modifiers to each unique combination of policies and claim types: OEM parts, outside parts and services, labor, travel, dealer freight duty and the total claim amount. You can create, search and maintain payment definitions in an easy and efficient manner.

 

To view the payment definition page,

  • Click on the Payment Definitions link under the Common Actions on your Home page.

  • Click on the Warranty Admin tab on the left-hand pane. The tab will expand to show all folders under it. Click on the Manage Claim Payment folder. All links under this folder will be shown. Click on the Configure Claim Payment Definition link.

Either of the above actions takes you to the Payment Definitions page. On the right-hand pane, you can see the Payment Definitions table that lists all definitions created by you. See Lists for more information on actions allowed on the Payment Definitions table.

 

The Payment Definitions table provides the following information about each definition:

  • Policy/Policy Label: Name of the policy or name of the label associated with a group of policies to which this definition is applicable.

  • Claim Type: Type of claim to which this definition is applicable.

Double click on the definition to view all details on a new page, <definition name>.

 

Creating a Payment Definition

You can create a new payment definition for each unique combination of policies and claim type and associate various modifiers to the definitions.

 

To create a new definition:

  • Click on the button.

  • You will see the New Payment Definition page. This page is divided into various sections.

In the Conditions section, enter the following details:

  • Policy Name: Name of the policy to whom the new payment definition is applicable. You can associate the new payment definition with a group of policies by clicking on the Specify Policy Label link. The Policy Name field now changes to Policy Label field and you can enter a group label here. To switch to the Policy Name field, click on the Specify Policy Name link. This field supports auto-complete.

  • Claim Type: Use this drop down option to specify various claim types to which the new payment definition is applicable. The options available in this field are: All, Parts, Machine, and Campaign.  You can select a particular claim type or select All to apply the payment definition to all claim types.

In the Payment Definition section, you can specify which modifiers are applicable to the policy/group of policies, at various levels, in payment computation.

  • OEM Parts: To apply modifiers on OEM parts reimbursement, click on the icon in this section. All modifiers created on OEM parts will be displayed. A text box is provided against each modifier. This text box allows you to specify the level at which a modifier will be applied during the OEM parts payment computation. Enter the level number in this field. If the level field is left blank, that modifier will not be applied during computation. To apply all selected modifiers during OEM part payment computation, check the OEM Parts check box.

  • Outside Parts/Services: To apply modifiers on outside parts/services reimbursement, click on the icon in this section. All modifiers created on non-OEM parts will be displayed. A text box is provided against each modifier. This text box allows you to specify the level at which a modifier will be applied during the non-OEM parts payment computation. Enter the level number in this field. If the level field is left blank, that modifier will not be applied during computation. To apply all selected modifiers during outside parts/services payment computation, check the Outside Parts/Services check box.

  • Labor: To apply modifiers on labor wages reimbursement, click on the icon in this section. All modifiers created on labor will be displayed. A text box is provided against each modifier. This text box allows you to specify the level at which a modifier will be applied during labor wages computation. Enter the level number in this field. If the level field is left blank, that modifier will not be applied during computation. To apply all selected modifiers during labor wages computation, check the Labor check box.

  • Travel: To apply modifiers on travel charges reimbursement, click on the icon in this section. All modifiers created on travel will be displayed. A text box is provided against each modifier. This text box allows you to specify the level at which a modifier will be applied during travel charges computation. Enter the level number in this field. If the level field is left blank, that modifier will not be applied during computation. To apply all selected modifiers during travel charges computation, check the Travel check box.

  • Dealer Freight Charge: To apply modifiers on freight charges claimed by dealer, click on the icon in this section. All modifiers created on freight charges will be displayed. A text box is provided against each modifier. This text box allows you to specify the level at which a modifier will be applied during freight charge computation. Enter the level number in this field. If the level field is left blank, that modifier will not be applied during computation. To apply all selected modifiers during freight duty compensation computation, check the Dealer Freight Charges check box.

  • Claim Amount: To apply modifiers on the total claim amount, click on the icon in this section. All modifiers created on claim amount will be displayed. A text box is provided against each modifier. This text box allows you to specify the level at which a modifier will be applied during claim amount computation. Enter the level number in this field. If the level field is left blank, that modifier will not be applied during computation. To apply all selected modifiers during claim amount computation, check the Claim Amount check box.

For example, consider the following scenario, you have 4 modifiers, M1, M2, M3 and M4 under the OEM Parts section. M1 is applied at level 2, M2 at level 1 and M3 at level 3. M4 is not applicable to the current payment definition. The payment expression for OEM parts will be: [{(Total OEM Parts Amount * M2) * M1} * M3].

 

Note: Click on the icon to hide the displayed modifiers.

  • After selecting the modifiers and setting a level for their application, click Continue.

  • New Payment Definition page now refreshes to display the various rules created by the application of different modifiers on each selected components of the claim.

  • Click Cancel if you want to close this page without saving any data entered by you.

  • Click Save. A success message is shown indicating the new definition is saved.

Note: The following are applicable to the levels:

  • Modifier at level 1 is applied to the base value, i.e., the total amount for that claim payment component. All modifiers at consecutive levels will be applied to the computed value of the previous level.

  • You can assign the same level to more than one modifier. Then all modifiers at the same level will be applied to the computed value of the previous level.

Editing a Payment Definition

You can edit a payment definition and reset the parameters to suit your needs.

 

To edit a definition:

  • Double click on a payment definition to open it in the detailed page.

  • You can edit any field in this page.

  • Click Cancel if you want to close this page without saving any changes made to the definition.

  • Click Save. A success message will be shown indicating all changes are saved.

  • To delete this definition, click Delete . A success message will be shown indicating the definition is deleted.

Defining Commercial Policy Payment Definition

When a claim is out of warranty, you file a commercial policy claim. The selection of commercial policy field while filing a claim will pick the set payment definition on the basis of which the different cost categories such as Miscellaneous Parts, Labor and Travel details, etc. will appear on the claim page.

 

To set the payment definition for Commercial Policy claims:

  • Click on Manage Claim Payment tab under Warranty Admin on the left-hand pane. This will reveal the sub-folders Create Claim Payment Modifier and Configure Claim Payment Definition. Click on Configure Claim Payment Definition.

  • Click on the button in the listing toolbar. This page lists various payment definitions for commercial policy claims. Each payment definition has a selected set of cost categories which will appear when you file a claim.

  • Select the payment definition you want to set for commercial policy claims by clicking on the radio button.

  • Click Update.

The selected payment definition will be set for all commercial policy claims.
