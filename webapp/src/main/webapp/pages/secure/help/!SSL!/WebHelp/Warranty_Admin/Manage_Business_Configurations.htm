MANAGE business conFIGURATIONS

The Warranty Management application allows you to create various business rules for Claims, Inventory, Supplier Recovery and Administration.

 

The various tabs present under the Claim section are Home Page, Claim Input Parameters, Claim Submission, Claim Processing, Return Part Management, Field Modification List, FOC Claim and Claim Display.

 

Click Home Page to display the various fields under this tab:

  • Internal Replies Auto Reply Window: Number of days after which a claim is automatically replied and returned to the processor.

  • Date to be Used for Draft Claim Deletion: Date based on draft to be deleted.

  • Days for Draft Auto Delete: Days for draft auto delete.

  • Days for Forwarded Claim Denied: Days after which forwarded claims are denied if not responded.

  • Stock Inventory Warning Window: Number of days from which a dealer is displayed a warning regarding the ageing of equipment in stock.

Click Claim Input Parameters to display the various fields under this tab:

  • Is End Customer Match Read Applicable: Match read check for claims.

  • Claim Types applicable for Claim Submission: The claim types which are enabled.

  • Is Dealer can select Multiple Job Codes: If a dealer can select multiple job codes.

  • Causal items on Claim Configuration: Selected item types will be allowed in claim entry as causal part.

  • Replaced items on Claim Configuration: Selected item types will be allowed in claim entry as part replaced.

  • Is filing of Non Serialized claim allowed: Determines if filing of Non Serialized claim is allowed

  • Is Invoice Number to be captured in Claim Submission: Determines if invoice number is to be captured.

  • Enable Over Time Labor Split: This configuration decides whether Labor Type Split should be enabled and shown for current business or not.

  • Is Date Code applicable for Claim submission: This configuration decides whether Date Code is applicable or not.

  • Is Parts Claim Without Host Allowed: Is Parts Claim Without Host Allowed

  • Capture Source Warehouse: Capture Source Warehouse

  • Is Failure Detail applicable for Claim submission: This configuration specifies whether dealers are allowed to see Failure Detail or not.

  • Enable Standard Labor Hours: Enable Standard Labor Hours.

  • Display Legal Disclaimer: Display Legal Disclaimer.

  • Use adjusted price from ERP remote system for claim calculation: Use adjusted price from ERP remote system for claim calculation.

  • Display Item Number for non-serialized claims: Display Item Number for non-serialized claims

  • Item types allowed on Parts Claim: Selected Item Types  will be allowed in Parts Claim

  • Can dealer search for other dealers retail inventory: This configuration specifies whether dealers are allowed to search for other dealers retail inventory or not.

  • If default processor needs to be shown to Dealer on all system generated audits: This configuration specifies whether default processor needs to be shown to Dealer on all system generated audits or not.

  • Can BU Part be replaced by non-BU Part: In a claims component Replaced section, can a BU part be replaced by a non-BU part.

  • Is alarm codes section visible: Specifies if alarm codes section is visible or not.

  • Display CP Flag (checkbox) on claim Page 1: Specifies whether to display CP Flag (checkbox) on claim Page 1 or not.

  • Display Part Serial Number field on Parts Installed/Removed section: Specifies whether to display Part Serial Number field on Parts Installed/Removed section or not.

  • Over Time Labor Split Distribution: This configuration decides whether Labor Type Split should be always inclusive, always exclusive or optional for dealer; as far as distribution is concerned for labor hours.

Click Claim Submission to display the various fields under this tab:

  • Denied Claim Re-submission Window Period: Window period within which the dealer can take action on denied claims.

  • Maximum Number of Claim Re-submissions allowed for a Dealer: Number of resubmissions allowed on a claim.

  • Partial Accepted Claim Re-submission Window Period: Window period within which the dealer can resubmit a partially accepted claim.

  • Default Rejection Reason for Claim Processing: Default rejection reason for Claim Processing.

  • Default Rejection Reason for Overdue Parts: Default rejection reason for Overdue Parts.

  • Default Rejection Reason for Forwarded Claim: Default rejection reason for Forwarded Claim.

  • Number of SMR Claim Re-submission allowed to a Dealer: Number of SMR Claim resubmits allowed to a Dealer

  • Currency to be used for ERP interactions: Currency to be used for ERP interactions

Click Claim Processing to display the various fields under this tab:

  • Default Review Responsibility while Processing a Claim: Default Review Responsibility

  • Default Acceptance Reason while Processing a Claim: Default Acceptance Reason

  • Yellow Warning Display Window on Service Home Page: Yellow Warning Display Window on Service Home Page

  • Days after which Red Warning is to be Displayed: Red Warning Display Windowon Service Home Page

  • External Replies Policy Window: Days within which the dealer has to reply

  • Internal Claim Aging Window: Indicates ageing of claims in days to Internal users.

  • Can Dealer Edit the Forwarded Claim: Specifies if a dealer can edit the Forwarded Claim.

  • Enable CP Advisor: This configuration decides whether CP Advisor should be enabled and shown for current business or not.

  • Default acceptance reason for CP while processing a claim: Default acceptance reason for CP.

  • Enable UOM in price calculation: Specifies whether to enable or disable UOM in price calculation

  • Capture Selling Entity: Specifies whether to capture selling entity or not.

  • Check Recovery Flag based on Review Responsibility: Specifies whether to check Recovery Flag based on Review Responsibility or not.

  • Consider Warranty Coverage for Part Claim: Specifies whether to consider Warranty Coverage for Part Claim or not.

Click Return Part Management to display the various fields under this tab:

  • Waiting Part Returns to be Denied If Not Shipped: Waiting period for Part Returns claims to be Denied If Not Shipped

  • Number of Days after which the parts marked shipped, if not returned will be acted automatically: Number of Days after which the parts marked shipped, if not returned will be acted automatically according to the action selected (move or auto deny).

  • Action to be taken on Parts marked shipped but not received in window: Action to be taken on claims with parts marked shipped but not received within the defined window.

  • Part Return Status to be considered for Max Quantity: Part Return Status to be considered for Max Quantity

  • Deny Shipment Generated Claims Crossed Window Period Days: Deny Shipment Generated Claims which have crossed the defined window period days.

  • Enable Dealers to view Part Return list: Specifies whether to enable dealers to view Part Return list or not.

Click Field Modification List to display the various fields under this tab:

  • Campaign - Days for Yellow Warning: Number of days from which yellow warning is displayed regarding the duration of a campaign.

  • Campaign - Days for Red Warning: Number of days from which red warning is displayed regarding the duration of a campaign.

Click FOC Claim to display the various fields under this tab:

  • Days for FOC Auto Submission: Days for FOC Auto Submission

  • Is Service provider number required for anonymous third party: Specifies whether the service provider number is required for anonymous third party or not.

Click Claim Display to display the various fields under this tab:

  • Show dealer full claim audit history: Specifies whether to show dealer full claim audit history or not.

  • Show dealer Actual Status of claim: Specifies whether to show dealer the actual status of claim or not.

  • Show dealer who claim is currently with: Specifies whether to show dealer with whom the claim is currently with or not.

The various tabs present under the Inventory section are WR/ETR Configuration, Warranty Registration, Transfers(ETR, D2D), and Search.

Click WR/ETR Configuration to display the various fields under this tab:

  • Enable Generic Attachment for Inventories: This configuration decides whether Generic Attachment should be enabled and shown for current business or not.

  • Is Additional information details applicable: This configuration decides whether additional information details are applicable or not.

  • Is Extended Warranty Price Check Enabled: This configuration decides whether Extended Warranty Price Check is enabled or not.

  • Is marketing information by product applicable: This configuration decides whether marketing information by product is applicable or not.

  • Capture Installing Dealer/Installation Date on WR/ETR: This configuration decides whether to capture Installing Dealer/Installation Date on WR/ETR or not.

  • Perform Dealer to Dealer Transaction on Warranty Registration: This configuration decides whether to perform Dealer to Dealer Transaction on Warranty Registration or not.

Click WR/ETR Configuration to display the various fields under this tab:

  • Date to be Used for Warranty Registration Modification/Deletion: Specify the date to be used for Warranty Registration Modification/Deletion.

  • Days to be Used for Warranty Registration Modification/Deletion: Configure the days to be used for Warranty Registration Modification/Deletion.

  • Warranty Registration needs Manual Approval: If set as Yes, Warranty Registration will require manual approval.

  • Customer Types Displayed in WR: Lists the customer types displayed in the drop down- menu on the Warranty Registration page.

  • Warranty Start date override window period by Installation Date: Period within which Warranty start Date will be overridden by the Installation Date if the Installation Report is submitted by the dealer.

  • Allow warranty registration across other dealers stock inventory: Specifies if the dealers are allowed to register units in other dealers’ stock or not.

Click Transfers(ETR, D2D) to display the various fields under this tab:

  • Customer Types Displayed in ETR: List the customer types displayed in the drop-down menu in the ETR page.

  • Is Manual Approval Flow required for ETR: Specifies whether manual approval flow is required for ETR or not.

Click Search to display the various fields under this tab:

  • Allow dealer to search based on Build Date: Specifies whether to allow dealer to search based on Build Date or not.

  • Allow dealer to search based on Manufacturing Site: Specifies whether to allow dealer to search based on Manufacturing Site or not.

  • Customer Types Supported in Quick Search for External Users: The list of external users who can search inventory that belongs to the selected Customer Types using the Quick Search functionality.

  • Enable Factory Order Number: This configuration decides whether Factory Order Number should be enabled and shown for the current business or not.

  • Field Modification Campaign Classes For Warning: The list of Field Modification Campaign Classes for Warning.

  • Can external user purchase extended warranty: This configuration decides whether external user can purchase extended warranty or not.

  • Date from which Extended Warranty Purchase will start: Date from which Extended Warranty Purchase will start.

  • Extended Warranty Purchase Logic Driven by: Date to be considered to check the availability of an Extended Plan on EWP.

  • Can external user perform Retail Machine Transfer: Specifies whether external user can perform Retail Machine Transfer or not.

  • Display additional end user and claim info on EHP: Specifies whether to display additional end user and claim info on EHP or not.

The Supplier Recovery tab is present under the Supplier Recovery section.

Click Supplier Recovery to display the various fields under this tab:

  • Debited Recovery Claims Ageing period: Debited Recovery Claims Ageing period.

  • Maximum Number of Dispute: Number of disputes allowed on a claim.

  • Cost Price Type Configuration: Cost Price Type Configuration.

  • Do serialized parts have shipment date: Used to check the coverage applicability of contracts defined by the Date of Manufacture.

  • Use Installed Parts for calculations on Recovery Claims: Used to configure if installed parts should be used to calculate recovery claims instead or removed parts. This is not applicable if Parts Replaced/Installed section is disabled.

The various tabs present under the Administration section are Modifiers, Warranty Plan and Miscellaneous.

Click Modifiers to display the various fields under this tab:

  • Customer Types for Base Rates and Modifiers: Base Rates and Modifiers can be setup only for the selected Customer Types.

  • Customer Types Displayed in Modifiers: Lists the customer types displayed in the drop down menu on the Modifiers page.

Click Warranty Plan to display the various fields under this tab:

  • Customer Types Displayed in Policy: Lists the customer types displayed in the drop down menu on the Warranty Plan Setup page.

Click Miscellaneous to display the various fields under this tab:

  • Whether to log the Login History of Users: Login History for Parts Collaboration.

  • Allow Session Time Out: Specifies whether to allow Session Time Out or not.

  • Type of inventories allowed for DCAP: Specifies type of entries for DCAP.

  • Is supplier required by ERP for cost fetch: Specifies whether supplier is required by ERP for cost fetch or not.

  • Roles To Be Displayed: Specifies the roles to be displayed.

  • Can create Dealer Group Code: Specifies whether Dealer Group Code can be created or not.
