Part Return Configuration

The OEM requires the Dealer to return certain failed parts for further inspection. These parts must be returned to specific locations to allow better management of part returns. The Warranty Management application allows you to configure various return criteria like return location, payment condition for claims involving part returns, days within which the part must be returned by the Dealer, etc. for each part marked for return.

 

The Part Return Configuration option allows you to define rules for returning replaced parts based on the following parameters: part number or part group name, Dealer or Dealer group name, type of claim, warranty type and product. You can define a part return configuration for each unique combination of these parameters. The criteria specified in these configurations cane be applied over a specified time period, defined by the repair date range. You can create, search and maintain part return configurations in an easy and efficient manner using the Part Return Configuration option.

 

To go to the part return configuration page,

  • Click on the Part Return Configuration link under Common Actions column on your Home page.

  • Click on the Warranty Admin tab on your left-hand pane. All folders under this tab are shown. Click on the Part Return Configuration folder.

Either of the above actions takes you to the Part Return Configuration page. On the right-hand pane, you can see the Part Return Configurations table that lists all configurations created by you. See Lists for more information on navigating through this table.

 

The Part Return Configurations table provides the following information about each configuration:

  • Part Number/Group Name: Serial number of the part or name of group of parts to which this configuration is applicable.

  • Dealer/Dealer Group: Name of the dealer or dealer group to which this configuration is applicable.

  • Claim Type: Type of claim to which this configuration is applicable.

  • Warranty Type: Warranty type to which this configuration is applicable.

  • Product: Products to which this configuration is applicable.

By default, details of the first configuration in the Part Return Configurations table are displayed in the preview pane. Click on a configuration to view its details in the preview pane. To view all details in a single page, double click on the configuration in the table. The Edit Part Return Configuration <n> page is shown.

 

The details in the preview pane and the detail page are shown under various sections, for ease of navigation. The following sections are shown:

  • Conditions: You can view the following details here - dealer name, type of claim, warranty type, products and the part number to which the rules in this configuration are applicable.

  • Part Return Configuration: You can view the following details here - manufacture date range, in the user specified format, payment condition, whether the part should be returned only if it is the causal part, return location and the days due within which the part must be returned.

You can create new part return configurations and edit existing configurations in the Part Return Configuration page.

 

Creating a Part Return Configuration

You can create a new part return configuration by specifying a unique set of values to the above mentioned parameters.

 

To create a new configuration:

  • Click on the option.

  • You will see the New Part Configuration page. This page is divided into various sections.

In the Conditions section, specify the following details:

  • Dealer: Name of the dealer to whom the new modifiers are applicable. You can associate the new modifier with a group of dealers by clicking on the Specify Dealer Group link. The Dealer field now changes to Dealer Group field and you can enter a group name here. To switch to the Dealer field, click on the Specify Dealer link. This field supports auto-complete.

  • Warranty Type: Use this drop down option to specify the type of warranty to which the modifier is applicable. The options available in this field are: All, Standard, Extended and Policy. You an select a particular warranty type or select All to apply the rates to all warranty types.

  • Claim Type: Use this drop down option to specify various claim types to which the new modifier is applicable. The options available in this field are: All, Parts, Machine, and Campaign.  You can select a particular claim type or select All to apply the rates to all claim types.

  • Product: Name of the product to which the new modifier is applicable. This field supports auto-complete.

  • Part Number/Group Name: Serial number of the part to which the new modifier is applicable. You can associate the new modifier to a group of items by clicking on the Specify Item Group link. The Part Number field now changes to Item Group field and you can enter a group name here. To switch back to the Part Number field, click on the Specify Part Number link. This field supports auto-complete.

In the Part Return Configuration section, you can specify rules to apply the modifier for specific manufacture date ranges:

  • Click on icon. This adds a row allowing you to specify the manufacture date range and the rate.

  • Manufacture Date: Use the option in the From and To fields to specify the start of the potential date range.

  • Only If Causal Part: You can specify whether the given part should be returned if it’s the causal part only or returned regardless of it being the causal part. Click on the appropriate radio button.

  • Return Location: Location where the part should be returned for inspection. This field supports auto-complete.

  • Days Due Within: Number of days within which the Dealer should return the replaced part to the specified return location.

  • You can add more rows and set more rules for different manufacture date ranges.

  • To delete a rule, click on icon corresponding to that row.

  • Click Cancel if you want to close this page without saving any data entered by you.

  • Click Save to save the new configuration. A success message is shown indicating the new configuration is saved.

Note: When creating rules for a configuration, you need to conform to the following:

  • To date cannot be before the From date.

  • From date can be a value placed in past, present or future.

  • From date can be a value placed in past, present or future.

  • If more than one rules are created, the date difference in the To date of one rule and the From date of the consecutive rule should be a single day.

  • You cannot assign overlapping values to From and To date fields.

Editing a Part Return Configuration

You can edit a part return configuration and reset the parameters to suit your needs.

 

To edit a configuration:

  • Double click on a configuration to open it in the detailed page.

  • You can edit any field in this page.

  • Click Cancel to close this page without saving any changes made to the configuration.

  • Click Save to save the changes made. A success message will be shown indicating all changes are saved.

  • To delete this configuration, click Delete. A success message will be shown indicating the configuration is deleted.

The part return configurations created here will be applied to the parts when they are replaced by service providers while repairing a failed truck.
