policy definition

A warranty policy is a legally binding document that details the coverage that the manufacturer offers on the product. A repair done on the product is considered under warranty only if it falls within the definition of the policy. The Warranty Management application allows you to define conventions and contract plan for a new policy. You can fix a priority for the new policy while setting its active period, applicability and registration terms.

 

To go to the Policy Definition page,

  • Click on the Policy Definition link under Common Actions column on your Home page.

  • Click on the Warranty Admin tab on your left-hand pane. All folders under this tab are shown. Click on the Manage Warranty Policies folder. All links under this folder are listed. Click on the Policy Definition link.

Either of the above actions takes you to the Policy Definition page. This page displays the Defined Policies table that lists all policies created by you.

 

The Defined Policies table provides the following information about each policy:

  • Policy Code: Identification code of the warranty policy.

  • Policy Name: Name of the warranty policy.

  • Warranty Type: Type of warranty offered by this policy.

  • Active From : Date from which this policy is available for registration.

  • Active Until : Date till which this policy is available for registration.

  • Status: Status of the policy.

  • Priority : Status established in the order of Importance or Urgency

Double click on a policy to view its details. For ease of navigation, the details of the policy are displayed under various sections. The following sections are displayed:

  • Policy Details: You can view the policy code, display name of the policy, warranty type, priority, active period, months covered from date of delivery of the truck to the end customer, and additional comments about the policy.

  • Applicability Terms: You can view all business conditions applicable to this policy.

  • Registration Terms: You can view all terms applicable while registering this policy and Products Covered by the policy.

  • Policy Audit history: You can view the policy history in this section like when and by whom it was last modified.

You can create new policies and edit existing policies in the Policy Definition page.

 

Creating a Warranty Policy

You can create a new warranty policy and define coverage details, registration details and applicability terms for it.

 

To create a new policy:

  • Click on option.

  • You will see the New Policy page. This page is divided into sections.

In the Policy Details section, provide the following details:

  • Policy Code: Identification code for the new policy.

  • Policy Name: Name of the policy.

  • NOMS Policy Option Code: Enter the NOMS Policy Option Code here.

  • NOMS Tier Description: Enter the NOMS Tier Description here.

  • Installer Certification Status: Certification status of the policy whether certified, not certified or any.

  • Warranty Type: Use this drop down option to select the type of warranty provided by this policy. The available options are: Standard, Extended and Policy.

  • Unit Hours/Energy Units Covered: Total unit hours covered by this policy.

  • Customer Type: Select the type of customer from the available options.

  • Build Date Applicable: Select this option, if desired.

  • Is Third Party Policy: Select this option, if desired.

  • Is Policy Applicable for DR: If you select the Warranty Type as Standard, this option is automatically selected.

  • Forced Selection Needed During DR: Select this option, if desired.

  • Months Covered from Date of Delivery: Number of months time for which the truck is covered from the date of delivery.

  • Months Covered from Original Delivery Date: Number of months time for which the truck is covered from the original date of delivery.

  • Months Covered from Build Date: Number of months time for which the truck is covered from the build date.

  • Months Covered from Shipment: Number of months time for which the truck is covered from the date of shipment.

  • Active From: Use the option to set the date from when this policy is available for registration. This date will be defaulted to the current date and you can edit it to any date.

  • Active Until: Use the option to specify the date till when this policy is available for registration. This date will be defaulted to

  • Comments: Any additional comments on this policy.

  • Priority: Precedence of this policy over other policies, when applicable to the same truck. The lower the number higher is the priority. The priority should be unique across all policies.

  • Terms and Conditions: Any additional terms and conditions on this policy.

In the Applicability Terms section, you can add business conditions for this policy:

  • Click on the icon to add a business condition to this policy. A pop-up window appears.

  • Search Business Conditions: Enter relevant data to filter the business conditions that you want to select.

  • Click Go. All business conditions matching the specified criteria will be displayed.

  • To select a business condition, check the check box against it. You can make multiple selections here.

  • Click Add Business Conditions. All conditions selected by you will be displayed in the Applicability Terms section.

  • You can remove an already selected condition by clearing the checkbox against it and clicking Submit.

In the Registration Terms section, you can add preconditions for registering and transferring this policy:

  • Products Covered sub-section:

  • Products/Models: The products or models to which this policy is applicable. This field should support all item groups. To add a product, click on the add icon . A new row will appear in this sub-section. Enter the product name in the field provided. This field supports auto-complete. To remove a product from this sub-section, click on the remove icon alongside that product.

  • Add Dealer/Dealer Group - In this section, you can add a Dealer or a Dealer Group.

  • To add a Dealer/Dealer Group, click  . This displays a pop-up window Add Dealer/Dealer Group.

  • Enter the Dealer Name and click Add. To add Dealer Group, click Dealer Group link and enter the Dealer Group name and click Add.

  • To cancel, click Cancel.

  • After entering details in all the sections, click Save. A success message is shown indicating the new policy is saved.

Editing a Warranty Policy

You can edit any parameters in a warranty policy and change its status at any given time.

 

To edit a policy:

  • Double click on a policy in the Defined Policies table, to open it in a new tab.

  • You can edit any field in this tab, except for the Policy Code field.

  • Click Cancel if you want to close this page without saving any changes made by you.

  • Click Save to save any changes made. A success message is shown indicating all changes are saved.

  • If you want to de-activate this policy, change the Active Until date to today’s date and click Save. The policy will become inactive from the next day.

Adding/Removing a Label

You can associate labels to each policy. The Warranty Management application allows you to add new labels and also associate predefined labels to multiple policies. Also, you can selectively remove associated labels from the policies.

 

To add a new label:

  • Click on a policy to select it. You can select multiple policies by pressing the Ctrl key and simultaneously clicking the policies.

  • Click on icon. A pop-up window is shown.

  • In the Label field, enter the name of the label that you wish to apply. This field supports auto-suggest for all previously created labels. You can also create a new label by entering its name in this field.

  • Click Select. The label name that you entered will be listed in the Selected Labels section.

  • You can add multiple label names in the same manner.

  • To remove a label from this list, enter the name of that label in the Label field and click Clear.

  • To apply the selected labels, click Apply Label(s). The selected labels will be applied to the selected policies.

To remove a label:

  • Click on a policy whose label you want to remove. You can select multiple policies by pressing the Ctrl key and simultaneously clicking on policies.

  • Click on icon. A pop up window is shown.

  • In the Label field, enter the name of the label that you wish to apply. This field supports auto-suggest for all previously created labels. You can also create a new label by entering its name in this field.

  • Click Select. The label name that you entered will be listed in the Selected Labels section.

  • You can add multiple label names in the same manner.

  • To remove a label from this list, enter the name of that label in the Label field and click Clear.

  • To remove the selected labels, click Apply Label(s). The selected labels will be removed from the selected policies.
