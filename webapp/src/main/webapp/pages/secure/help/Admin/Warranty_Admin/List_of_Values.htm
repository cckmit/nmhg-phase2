List of values

At various points in the application, we have drop down lists to select from. The several options available under these categories are not immutable i.e. they are changeable, and the administrator may wish to add or remove any value. These drop-down lists are called list of values in the system. You can edit and search for various LOVs, add new values and activate/inactivate existing values in various lists.

 

The warranty application has a pre-defined set of list of values. The application allows the administrator to add, update or deactivate a value within this pre-defined set. Each LOV set has a name that identifies its usage in the application. At present the sets available in the application along with their usage are:

  • SMR Reason: This set captures the list of reasons that reflect why the dealer is requesting a warranty claim for service manager review.

  • Claim Acceptance Code: This set captures the list of reasons or codes for accepting a warranty claim by the claims processor.

  • Claim Denial Code: This set captures the list of reasons or codes for denying a warranty claim by the claims processor.

  • Internal Inspection Result: This set captures the list of results that the warehouse inspector can record on inspecting a part.

  • Supplier Action Reason: This set captures the list of reasons on the action taken by the supplier for a part being recovered.

  • Supplier Inspection Result: This set captures the list of results that the supplier can record on inspecting the part

  • Service Campaign Status Reason for Serial Number: This set captures the list of reasons that the warranty administrator can record while updating the campaign status for each serial number.

To view the LOVs, click on the Warranty Admin tab on your left-hand pane. All folders under this tab are shown. Click on the List of Values folder. On this page you can see the LOV table that lists all LOVs in the Warranty Management application.

 

On double clicking the LOV category, you can view the following information about each value:

  • Code: Identification label of the value.

  • Description: A short description about the value.

  • State: State refers to whether a value is in use, i.e., Active, or not in use, i.e., Inactive in the list that it is associated with.

Double click on the list to view all values associated with it, on a new page, <LOV name>. You can add or delete the values belonging to the selected list on the <LOV name> page.

 

Adding a Value

You can create a new value to a list by specifying a unique identification label to it.

 

To create a new value:

  • Double click a list. The <LOV name> page is shown.

  • Click on option.

  • You will see the New Value page.

  • Code: Enter an identification label for the new value.

  • Description: Enter a short description about the new value.

  • If you want to close this page without saving any data entered by you, click Cancel.

  • To add the value to the LOV, click Save.

The newly created value will be shown in the Values table of the corresponding LOV.

 

Editing a Value

You can edit a value and change its status anytime.

 

To edit a value:

  • Double click on a value in the Values table, to open it in a new page.

  • You can edit any field in this page.

  • If the value is currently in the active state, and you want to deactivate it, click on Deactivate. Similarly, if a value is currently deactivated, you can activate it by clicking Active.

  • Click Cancel if you want to close this page without saving any changes made by you.

  • Click Save. A success message will be shown indicating all changes are saved.

  • To deactivate this value, click Deactivate.

A success message will be shown indicating the value is deactivated.

Note: To activate a deactivated value, click on the Activate button.
