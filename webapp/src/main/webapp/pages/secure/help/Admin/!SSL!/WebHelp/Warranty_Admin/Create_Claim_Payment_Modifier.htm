CREATE CLAIM PAYMENT MODIFIER

A claim amount consists of various components such as cost of replaced parts, cost for labor and travel involved and other miscellaneous expenses. Payment modifiers allows you to alter the claim amount either through discounts or surcharges for each of these components.

 

You can create payment modifiers for the following components of a claim payment: parts and Outside parts/services, labor, travel, dealer freight charges and the total claim amount. Payment modifiers can be defined for unique combinations of the following parameters: dealer/dealer group, warranty type, claim type and product. You can create, search and maintain payment modifiers in an easy and efficient manner using the Payment Modifiers option.

 

To view the payment modifiers page,

  • Click on the Create Claim Payment Modifier link in the Warranty Admin section under Folders column on your Home page.

  • Click on the Warranty Admin tab on the left-hand pane. The tab will expand to show all folders under it. Click on the Manage Claim Payment folder. All links under this folder will be shown. Click on the Create Claim Payment Modifier link.

Either of the above actions takes you to the Payment Modifiers page. On the right-hand pane, you can see the following sections: Create Modifier Label and Choose Existing Modifier Label.

 

The Create Modifier Label section allows you to create a new payment modifier label.

 

To create a new label:

  • Modifier Label: Enter a label for the payment modifiers. This is a tag assigned to a set of payment modifiers which are applicable to a particular section.

  • Section: Use the drop down option to select a section to which the payment modifiers are applicable. The options available in this drop down are: OEM Parts, Non-OEM Parts, Labor, Travel, Item Freight Duty, Meals, Parking and Claim Amount.

  • Click on Create. The new modifier label is created and will be displayed in the list of existing payment modifiers.

The Choose Existing Modifier Label section lists all existing payment modifier labels. It also allows you to define new payment modifiers for these labels.

 

If you want to create new modifiers for a label, click on the Set Values link, corresponding to that label, in the Actions column. This will open a new page, <modifier label>.

 

Creating a Payment Modifier

The <modifier name> page displays all modifiers associated with the selected payment modifier label, in the Payment Modifier table.

 

The Payment Modifier table provides the following information about each modifier:

  • Dealer/Dealer Group: Name of the dealer or the dealer group to which this payment modifier is applicable.

  • Claim Type: Type of claim to which this payment modifier is applicable.

  • Warranty Type: Warranty type to which this payment modifier is applicable.

  • Product: Products to which this payment modifier is applicable.

You can create a new payment modifier by specifying a unique set of values to these parameters.

 

To create a new modifier:

  • Click on the button.

  • You will see the New Payment Modifier page. This page is divided into various sections.

In the Conditions section, specify the following details:

  • Dealer: Name of the sealer to whom the new payment modifier is applicable. You can associate the new payment modifier with a group of sealers by clicking on the Specify Dealer Group link. The Dealer field now changes to Dealer Group field and you can enter a group name here. To switch to the Dealer field, click on the Specify Dealer link. This field supports auto-complete.

  • Product: Name of the product to which the new payment modifier is applicable. This field supports auto-complete.

  • Warranty Type: Use this drop down option to specify the type of warranty to which the payment modifier is applicable. The options available in this field are: All, Standard, Extended and Policy. You an select a particular warranty type or select All to apply the payment modifier to all warranty types.

  • Claim Type: Use this drop down option to specify various claim types to which the new payment modifier is applicable. The options available in this field are: All, Parts, Machine, and Campaign.  You can select a particular claim type or select All to apply the payment modifier to all claim types.

In the Rates section, you can specify rules to apply modifier denominations for specific validity date ranges:

  • Click on icon. This adds a row allowing you to specify the repair date range and the rate.

  • Validity Date: Validity date range is the time over which this modifier is active and will be applicable. Use the option in the From and To fields to specify the start of the potential date range. The date will be displayed in the format selected by you.

  • Percentage: Specify the applicable percentage modifier in this field. You can specify a positive or a negative value in this field, depending on whether the modifier is a discount or a surcharge.

  • You can add more rows and set more rules for different date ranges.

  • To delete a rule, click on icon corresponding to that row.

  • If you want to close this page without saving any data entered by you click Cancel.

  • Click Save. A success message is shown indicating the new modifier is saved.

Note: When creating rules for a payment modifier, you need to conform to the following:

  • To date cannot be before the From date.

  • From date can be a value placed in past, present or future.

  • To date should be a value placed in future.

  • If more than one rules are created, the date difference in the To date of one rule and the From date of the consecutive rule should be a single day.

  • You cannot assign overlapping values to From and To date fields.

Editing a Payment Modifier

You can edit a payment modifier and reset the parameters to suit your needs.

 

To edit a modifier:

  • Double click on a payment modifier to open it in a detailed page.

  • You can edit any field in this page.

  • Click Cancel if you want to close this page without saving the changes made to the modifier.

  • Click Save. A success message will be shown indicating all changes are saved.

  • To delete this modifier, click Delete . A success message will be shown indicating the modifier is deleted.

Note: You can edit the To date and place it in the past.
