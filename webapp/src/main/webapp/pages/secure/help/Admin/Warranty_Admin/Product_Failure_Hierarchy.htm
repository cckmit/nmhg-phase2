Product Failure Hierarchy

Failure hierarchy is a tree structure which allows easy identification of the exact location of failure in an truck. A failure hierarchy is uniquely defined for a particular product/model of the truck. It lists each component of the truck in a manner which allows easy mapping of job codes. You can also assign a tread bucket to each component.

 

The hierarchy resembles a folder-like structure which indicates the relationships between the assemblies that make up that product. For example, a product could consist of the following setup (part of):

• Electrical (E1)

o Engine (001)

- Alternator (005)

- Battery (010)

- Fuse (034)

o Controller (023)

- Tilt Frame (048)

• Hydraulic (H1)

o Brake (092)

- Pedal (167)

- Part Brake Lever (267)

o Cylinder (082)

- Steering (278)

- Tilt Frame (475)

• Hydrostatic (H2)

o Motor (057)

o Pump (037)

- Mount (193)

 

The above setup indicates that this product is made up of the structure as indicated above and if there is a fault in Brake Pedal, the fault location would be “Hydraulic – Brake – Pedal” / H1 – 092 – 167. By using this code, the exact location of the fault can be indicated.

 

The Warranty Management application allows you to define the failure hierarchy for each product/model of the truck. You can also define job codes for each component of the truck. You can create and maintain failure hierarchy and job codes in an easy and efficient manner.

 

To view the failure structure page,

  • Click on the Product Failure Structure link in the Warranty Admin section under the Folder column on your Home page.

  • Click on the Warranty Admin tab on your left-hand pane. All folders under this tab are shown. Click on the Manage Failure Structure folder. All links under this folder are shown. Click on the Product Failure Structure link.

Either of the above actions takes you to the Product Failure Hierarchy page.  On the right-hand pane, you can see the Failure Hierarchy table which allows you to create the failure hierarchy tree.

 

To create a failure hierarchy:

  • On the Product Failure Hierarchy page, use the Product/Model drop down option to select a product/model for which you want to create the failure hierarchy. This drop-down option lists all products/models of all trucks manufactured by the OEM.

  • In the first column, you can add a system or an attachment name.

  • Click on the icon alongside the first column header. A new row appears.

  • Click on the second cell in this row. A drop down option appears. Use this drop down option to select a system name. You can also manually enter a system name in this field.

  • When you create a new system name, the Create New Definition pop-up window appears, requesting you to authenticate the new system. Click Create to add the new system name to the existing drop down. To discard the new system, click Cancel.

  • A code is associated with each system name. This alphabetical code is auto-generated by the system.

  • You can add child nodes, consisting of sub-systems, to each system in the hierarchy. To add a child node, click on the icon in the first column, corresponding to the parent system. A new row appears.

  • Click on the second cell in this row. A drop down field appears. Use this drop down option to select a sub-system. You can also manually enter a sub-system name in this field.

  • A code is associated with each sub-system. This 3-digit numerical code is auto-generated by the system.

  • You can create child nodes under each sub-system by clicking the icon corresponding to that sub-system.

  • The Warranty Management application allows you to span this tree structure to four levels.

  • Job column: You can add a service procedure to each system or its sub-system.

  • To add a service procedure to a system/sub-system, click on the icon in the Job column, corresponding to that system/sub-system. A new row appears.

  • Click on the fourth cell in this row. A drop down field appears. Use this drop down option to select a service procedure. You can also manually enter a service procedure in this field.

  • When you create a new service procedure, the Create New Definition pop-up window appears, requesting you to authenticate the new service. Click Create to add the new service procedure to the existing drop down. To discard the new service procedure, click Cancel.

  • A code is associated with each service procedure. This alphabetical code is auto-generated by the system.

  • For Campaigns: Check this check box to associate a service procedure to a campaign.

  • Labor Hours: Enter the number of hours required to perform this service procedure.

  • You can add multiple service procedures for each system/sub-system.

  • Tread column: You can associate any system to an appropriate tread bucket, provided you have defined a service procedure for that system/sub-system. To associate a system/sub-system to a tread bucket, click on the Tread cell corresponding to the system/sub-system. A drop down field appears. Use this drop down option to select a tread bucket for the system/sub-system. The Last Updated column reflects the last date when the tread bucket was updated.

  • To delete a row click the icon corresponding to that row.

  • If you want to close this page without saving any details entered by you, click Cancel.

  • Click Save. A success message is shown indicating the new hierarchy is saved.

Note: The following points hold good while creating a hierarchy:

  • Every sub-system must have at least one service procedure associated with it. If a system has no child nodes, then at least one service procedure must be associated with that system.

  • Click on the icon corresponding to a system,  to hide the sub-systems, service procedures and treads associated with the system. To show the hidden details, click on the icon.

You can also edit an existing failure hierarchy for a product/model.

 

To edit a failure hierarchy,

  • Select the product/model whose failure hierarchy you want to edit, from the Product/Model drop-down.

  • The page refreshes to display the failure structure created by you.

  • Make any changes you wish to the structure.

  • Click Cancel if you wish to close this page without saving any changes made by you.

  • Click Save to save the changes made by you. A success message is shown indicating the changes made by you are saved.
