INTRODUCTION

The Warranty Management application is used by the various users:

  • Dealers use this application to file claims, register and transfer warranties, keep a record of all trucks in the stock and retail, and to manage part returns.

  • Processors and Service Managers use this application to verify the claims submitted by the dealers.

  • Receivers, Inspectors, and Part Return Administrators use this application to manage part returns.

Thus, it becomes essential for an authority to set up basic accessibility to all these users. The Warranty Administrator is a personnel who is permitted to configure certain features of the application accordingly to suit the various users.

 

A Warranty Admin can perform the following operations using the Admin module of the application:

  • Define new policies and manage existing policies

  • Establish flat rates for labor and travel

  • Create modifiers for item prices, claim payment, etc.

  • Configure failure hierarchy, job codes, fault found and caused by for various models of the truck

  • Create business rules for various modules of the application

  • Create and manage service campaigns

  • Configure part returns and warehouses and manage shippers

  • Create list of values for various drop-down menus that appears throughout the application

  • Upload end customer address in bulk

  • Create and maintain item, dealer and user labels

  • Create and maintain dealer and item groups

  • Create and submit claims on behalf of service providers

  • Register warranties for various trucks and generate truck transfer reports

  • Process claims submitted for manual adjudication

  • Maintain system settings

This user manual details all functions that can be performed by an Administrator, in a detailed manner.
