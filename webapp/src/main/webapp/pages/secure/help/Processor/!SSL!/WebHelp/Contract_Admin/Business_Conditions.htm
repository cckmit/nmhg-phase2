business conditions

The Warranty Management application allows you to create a pool of business conditions with single or a set of conditions under them. These business conditions can be used while creating various business rules.

 

To view the business conditions, click on the Contract Admin tab on the left-hand pane. Click on Manage Business Conditions. On the right-hand pane, you can see the Business Conditions table that lists all business conditions created by you. To know more about navigating through this table, see Lists.

 

To view the details of any condition in the preview pane, click on that condition. Double click on a condition to view it in the detailed page. You can create new conditions or edit existing conditions from the list.

 

Creating a Business Condition

You can create a new condition by specifying various claim parameters.

 

To create a new condition,

  • Click on the button in the listing toolbar.

  • The New Business Condition page is shown.

On this page, you can create several conditions and save them under a single business condition. You can also create a set of conditions and apply negation on them.

 

In the Provide a phrase describing the business condition, enter a name under which you want to save the business condition for future searches.

 

Using the Not operator,

 

The Not operator allows you to negate a condition.

 

 

The conditions created under one phrase can be applied in different ways.

  • Keep the Not check-box un-checked. Select the option All of the conditions are true in the drop-down. When a business condition with this combination is used in a business rule, the rule will be executed only if all the expressions in the business condition are satisfied.

  • Check the Not check-box. Select the option All of the conditions are true in the drop-down. When a business condition with this combination is used in a business rule, the rule will be executed if any one of the expressions in the business condition is not satisfied.

  • Keep the Not check-box un-checked. Select the option Atleast one of the conditions below is true in the drop-down. When a business condition with this combination is used in a business rule, the rule will be executed only if any one of the expressions in the business condition is satisfied.

  • Check the Not check-box. Select the option Atleast one of the conditions below is true in the drop-down. When a business condition with this combination is used in a business rule, the rule will be executed only if none of the expressions in the business condition are satisfied.

Every condition is made up of three parts, <attribute> <object> <value>.

The attribute is a property of the business object, i.e. the warranty claim. An operator is a mathematical operation used as part of the condition. The value part of the search condition indicates the operand against which the operator should be applied to the attribute. You will need to create a search condition using the attributes, objects and values provided in the query builder to search for claims.

 

Enter the following details to create a new condition:

  • In the Create Business Condition section, click on the option. A new row will be added.

  • Use the drop-down to select an attribute for your condition. For example, select Date of Repair from this drop-down.

  • Use the drop-down to select an object for your condition. The options in this drop-down depends on the attribute selected by you. For example, for the Date of Repair attribute, select is after from this drop-down.

  • Now you need to enter a value in the text field. This is a free text field. For example, use the option to select a date for this field.

  • You can add multiple conditions under the same phrase by clicking on the option.

  • You can delete a condition by clicking on the corresponding to that row.

  • If you wish to save this business condition, then click Save.

  • If you want to clear the options selected by you, click Reset.

For example, your condition will now be, Date of Repair is after 01-01-2008.

 

Nested conditions

Some of the claim parameters allow you to create nested conditions.

 

You can either select an object for the main attribute or add nested conditions with attributes, objects and values which correspond to the main attribute.

 

For example, for the attribute Causal Part, you can create an expression, Causal Part Item Number is exactly Item1.

 

You can also append an existing condition to a new condition.

 

To append an existing condition,

  • Click on the Choose Existing Condition(s) link. A pop-up window is shown.

  • In the Search Business Conditions field, enter a search criteria based on which you wish to search for existing conditions.

  • Click Go. All conditions with the specified search criteria will be listed on the pop-up window.

  • Select the check-box against the business condition which you want to append. You can select multiple conditions.

  • Click Add Business Conditions button. The selected conditions are appended to your new condition.

Edit Search

You can edit a business condition by opening it in the detailed page.

  • Double click on a business condition in the Business Conditions table. The selected condition will open in the detailed page.

  • Edit any parameter on this page.

  • Click Update to save the changes made by you.

You can delete a condition by opening it in the detailed page and clicking the Delete button. The condition will be deleted from the Business Conditions table.
